## 一、先回答你的问题：I2C的.c文件是什么？

**对！** I2C的.c文件就是**STM32作为I2C主机，发送和接收数据的底层驱动**。

MPU6050通过I2C协议和STM32通信，所以你需要：
1. **I2C驱动**（底层）：负责发送起始信号、发送地址、发送字节、接收字节、发送ACK/NACK、停止信号
2. **MPU6050驱动**（上层）：利用I2C驱动，去读/写MPU6050内部的寄存器

你看到的代码就是**MPU6050驱动**，它调用I2C驱动的函数来完成通信。

## 二、整体思路：先做什么，再做什么？

```
第1步：初始化I2C硬件
    ↓
第2步：用I2C读写MPU6050寄存器（底层通信）
    ↓
第3步：配置MPU6050内部寄存器（初始化）
    ↓
第4步：循环读取数据（加速度/陀螺仪/温度）
```

## 三、逐层拆解你的代码

### **第1层：I2C读写函数（你写的）**

MPU6050内部有很多**寄存器**，每个寄存器有地址。要读取数据，必须：
1. 告诉MPU6050"我要写"（假写），把寄存器地址发过去
2. 告诉MPU6050"我要读"，然后读数据

#### **读1个字节的时序（你代码里的）**
```c
Int_MPU6050_ReadByte(uint8_t reg_addr, uint8_t *receive_byte)
{
    起始信号();
    发送地址+写(0xD0);      // 告诉MPU6050：我要写
    发送寄存器地址(reg_addr); // 告诉MPU6050：我要读哪个寄存器
    起始信号();              // 重启总线
    发送地址+读(0xD1);      // 告诉MPU6050：我要开始读了
    读取1字节数据();        // 读数据
    发送NACK();             // 告诉MPU6050：读完了，别再发了
    停止信号();
}
```

**关键点**：为什么有两次起始信号？因为：
- 第一次：把"要读哪个寄存器"告诉MPU6050（这是写操作）
- 第二次：把"我要开始读了"告诉MPU6050（这是读操作）

这就是你注释里问的"**假写**"——第一次发送地址+写，但实际上不是写数据，而是写寄存器地址。

### **第2层：寄存器配置（初始化）**

MPU6050有几十个寄存器，你需要配置：
- **电源管理**：唤醒芯片
- **量程设置**：陀螺仪±2000°/s，加速度±2g/±4g/±8g/±16g
- **采样率**：多久采集一次数据
- **滤波设置**：低通滤波

你的`Int_MPU6050_Init()`做的就是这些：

```c
void Int_MPU6050_Init(void)
{
    1. 初始化I2C硬件           // 让STM32的I2C引脚能工作
    2. 复位MPU6050             // 写0x80到电源管理寄存器
    3. 唤醒MPU6050             // 写0x00到电源管理寄存器
    4. 设置陀螺仪量程          // 写寄存器MPU_GYRO_CFG_REG
    5. 设置加速度量程          // 写寄存器MPU_ACCEL_CFG_REG
    6. 关闭不需要的功能         // 中断、FIFO、第二I2C都关掉
    7. 配置时钟源              // 用陀螺仪X轴时钟，精度高
    8. 设置采样率              // 调用SetGyroRate
    9. 验证ID                  // 读寄存器0x75，应该是0x68
}
```

### **第3层：读取数据（还没写）**

初始化完成后，你需要循环读取数据：
1. 读加速度计寄存器（0x3B开始，6个字节）
2. 读陀螺仪寄存器（0x43开始，6个字节）
3. 读温度寄存器（0x41开始，2个字节）

```c
// 伪代码
void Read_MPU6050(void)
{
    uint8_t data[14];
    Int_MPU6050_ReadBytes(MPU_ACCEL_XOUT_H_REG, data, 14);
    
    // 解析数据
    int16_t ax = (data[0] << 8) | data[1];
    int16_t ay = (data[2] << 8) | data[3];
    int16_t az = (data[4] << 8) | data[5];
    int16_t temp = (data[6] << 8) | data[7];
    int16_t gx = (data[8] << 8) | data[9];
    int16_t gy = (data[10] << 8) | data[11];
    int16_t gz = (data[12] << 8) | data[13];
}
```

-

**核心理解**：MPU6050就是一堆寄存器，你要做的就是：
- **写寄存器**：配置它怎么工作
- **读寄存器**：获取传感器数据

I2C就是通道，读/写函数就是操作寄存器的工具。
